2026-07-06 — M0: SQLite store with migrations for the data model
- Added rusqlite 0.32 (feature "bundled" — vendors SQLite, no system dep) to the
  workspace deps and the loopfleet-store crate.
- crates/store/src/migrations/0001_init.sql: full initial schema matching the PRD
  data model — projects, plans, tasks (PK = plan_id + anchor normalized_text,
  line_hint as tiebreaker), runs (Model B: plan_id + task_anchor FK to tasks),
  iterations, sessions (M5 schema defined now), events (single append-only log,
  AUTOINCREMENT seq). Derived TaskStatus is intentionally NOT stored. Indexes on
  events(owner,seq), runs(plan_id,task_anchor), plans(project_id).
- crates/store/src/lib.rs: migration runner. MIGRATIONS is an ordered
  (name, include_str! sql) list; migrate() creates schema_migrations, applies
  each pending migration in its own BEGIN/COMMIT with ROLLBACK on error, skips
  already-applied ones (idempotent). open(path) opens a connection, enables
  foreign_keys pragma, and migrates. ":memory:" supported for tests.
- Tests (4, all passing): full schema present after open, migrate idempotent,
  foreign keys enforced (plan -> missing project rejected), events.seq
  autoincrements.

Design notes:
- Migrations are append-only and keyed by name; never edit or reorder a shipped

2026-07-06 — M1: Normalized event enum (adapter-sourced vs app-sourced lanes)
- crates/core/src/event.rs: NormalizedEvent, the single vocabulary every
  downstream consumer speaks. Variants match the PRD exactly: adapter-sourced
  lane (TurnStarted, AssistantText { text }, Reasoning { text }, ToolCall
  { call_id, name, input_excerpt }, ToolResult { call_id, ok, output_excerpt },
  CommandRun { cmd, exit: Option<i32> }, TurnCompleted { usage }, NeedsApproval,
  Failed { reason }, Ended) and the app-sourced lane (FileChanged { path }).
  ToolCall/ToolResult correlate by call_id (a shared String field on both).
- Lane enum { Adapter, App } + NormalizedEvent::lane(): only FileChanged is
  App; everything else is Adapter. The lane distinction is expressed as a method
  rather than splitting the type, so downstream still consumes one enum.
- Usage { input_tokens, output_tokens }: u64, Default, unreported counts = 0.
- Serde: internally tagged `#[serde(tag = "kind", rename_all = "snake_case")]`,
  so the store's normalized_event_json column holds e.g.
  {"kind":"tool_call","call_id":"…",…} and unit variants {"kind":"ended"}.
  All variants are unit or struct (no tuple variants), which internal tagging
  requires.
- NeedsApproval kept fieldless per PRD (only fires in M5 interactive sessions;
  headless never surfaces it — the sandbox is the boundary). No speculative
  fields added; revisit if M5 needs correlation.
- Placement: core, per core/lib.rs's own doc comment ("the normalized event
  enum" lands here). Re-exported from lib.rs (Lane, NormalizedEvent, Usage).
  store stays agnostic — it persists opaque JSON text, not the Rust type.
- Added serde dep to core, serde_json to core dev-deps + workspace deps
  (adapters will need serde_json for their stream mapping in the next M1 tasks).
- Tests (4, all passing): round_trips_every_variant (every variant survives
  JSON round-trip), kind_tag_shape_is_stable (the snake_case discriminator),
  tool_call_and_result_correlate_by_call_id, lanes_classify_correctly.

2026-07-06 — M1: AgentAdapter trait + stub replay adapter
- crates/adapters/src/lib.rs: the AgentAdapter trait, object-safe via async_trait
  so the M3 supervisor can hold a Box<dyn AgentAdapter> chosen by agent name.
  start_run(&RunSpec) -> RunHandle; open_session(&Path, SessionSeed) ->
  SessionHandle stays in the frozen signature but v1 impls return
  AdapterError::SessionsUnsupported (interactive sessions are M5).
  - RunSpec { cwd, prompt }: only what the stub + supervisor need in M1; per-agent
    flag sets / model land as real adapters do. SessionSeed { plan_file } present
    so the signature is frozen now, consumed by no v1 adapter.
  - RunHandle / SessionHandle wrap a tokio mpsc::Receiver<NormalizedEvent>. Bounded
    channel = the backpressure contract (matches the later event-log-writer task):
    a slow consumer stalls the producer, no unbounded buffer. Process-group
    ownership + stop/SIGTERM stay with the M3 supervisor, not the handle.
  - AdapterError { Spawn(io), Protocol(String), SessionsUnsupported } + Display /
    Error. Derived Debug on the handles so tests can unwrap_err.
- crates/adapters/src/stub.rs: StubAdapter — replays a canned event log, no
  process/network, deterministic. This is the fixture all UI work builds against
  before a real adapter exists. new(Vec<NormalizedEvent>) or from_jsonl(&str)
  (one event per non-empty line, same {"kind":…} shape the store persists; blank
  lines skipped, malformed JSON is a hard error). start_run clones the events and
  tokio::spawns a task that sends them in order over the channel, stopping if the
  receiver is dropped. open_session returns SessionsUnsupported.
- crates/adapters/fixtures/basic_run.jsonl: a realistic 9-event run (turn_started
  → assistant_text/reasoning → correlated tool_call/tool_result → command_run →
  turn_completed → ended) for the from_jsonl loader and UI reuse.
- Deps: added tokio (rt, sync, macros) and async-trait to workspace deps; adapters
  now depends on loopfleet-core, async-trait, tokio, serde_json. First async +
  tokio in the tree — foundational for the supervisor/event-log work, not
  speculative (PRD's trait is async; Tauri v2 is tokio-based).

Verification:
- CARGO_HOME=/tmp/loopfleet-cargo cargo test -p loopfleet-adapters — 5 passed

2026-07-06 — M1: Claude Code adapter, headless
- Captured ground truth first: ran the real `claude 2.1.201` binary with
  `-p … --output-format stream-json --verbose --dangerously-skip-permissions`
  in a throwaway git repo to see the actual line shapes, rather than mapping from
  memory. Line types observed: system/init, assistant (content blocks: thinking,
  text, tool_use), user (tool_result), result/success, plus noise (system
  hook_started/response, rate_limit_event).
- crates/adapters/src/claude.rs: ClaudeAdapter (stateless) + ClaudeMapper
  (stateful over one run's stream).
  - start_run spawns `claude` in spec.cwd, stdin null, stdout/stderr piped, and
    tokio::spawns drive() which reads stdout line-by-line, maps each via
    ClaudeMapper, and forwards over the same bounded mpsc(64) RunHandle path the
    stub uses (backpressure contract preserved). Receiver-dropped → start_kill +
    stop. Stream ending without a terminal `result` → synthesize
    Failed{reason: stderr} + Ended so consumers always see termination.
  - Mapping: system/init → TurnStarted. assistant blocks: text → AssistantText
    (empty skipped), thinking → Reasoning (empty skipped), tool_use → CommandRun
    when name=="Bash" (cmd from input.command, exit:None — Claude's bash result
    carries no numeric exit code) else ToolCall{call_id=id, name, input_excerpt}.
    user tool_result → ToolResult{call_id=tool_use_id, ok=!is_error,
    output_excerpt}, EXCEPT results of Bash calls are dropped (already normalized
    to CommandRun, which the enum carries with no result pairing — mapper tracks
    bash_calls ids). result → TurnCompleted{usage:{input_tokens,output_tokens}}
    on success or Failed{reason:result} on error, always followed by Ended.
    Unknown line types + blanks → ignored.
  - Excerpts truncated to 2000 bytes on a char boundary (event log stores
    excerpts, not full payloads). tool_result content flattened whether it's a
    plain string or an array of text blocks.
- crates/adapters/fixtures/claude_stream.jsonl: a 9-line fixture distilled from
  the real capture (init trimmed to the fields the mapper reads; real toolu_ ids,
  real Read/Bash tool_use + results, real usage numbers, a rate_limit_event noise
  line). The mapper's primary test runs against this.
- Cargo.toml: adapters tokio now enables process + io-util; dev-deps add
  tokio rt-multi-thread + tempfile "3" for the live test.
- Tests (10 mapper/unit + 1 ignored live): captured-stream mapping (asserts the
  full expected sequence, Read pair correlates, Bash→CommandRun with its result
  dropped, exactly one ToolResult, usage parsed), thinking→Reasoning &
  text→AssistantText, empty blocks skipped, non-Bash tool_use excerpt, error
  tool_result → ok:false, error result → Failed+Ended, noise/blank ignored,
  invalid JSON → Protocol error, array content flattened, excerpt char-boundary
  truncation. Plus live_run_against_fixture_repo (#[ignore]) which spawns the
  real CLI in a temp git repo — exercises spawn+drive end to end.

Design notes:
- CommandRun for Bash carries exit:None on purpose: Claude's bash tool_result
  exposes is_error (bool) but no numeric exit code, and the enum's exit is the
  process exit code — fabricating Some(0)/Some(1) would misrepresent it. The
  bash output itself is not representable (CommandRun has no output field); that
  is the PRD's normalization, and FileChanged (app lane) will catch effects.
- The mapper is stateful (bash_calls set) and fed one line at a time, so drive()
  stays a thin read-loop and the mapping is unit-testable with zero process.
