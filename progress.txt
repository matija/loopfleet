2026-07-06 — M1: Event-log writer (single-writer SQLite via a bounded channel)
- crates/store/src/event_log.rs: EventLog — the one place rows are appended to
  the `events` table. spawn(conn, capacity) takes ownership of the Connection and
  drains a bounded tokio mpsc of LogEntry on a tokio::task::spawn_blocking loop
  (rx.blocking_recv()), inserting each row. SQLite is single-writer, so funneling
  every producer through one writer means concurrent runs never contend for the
  write lock. The bounded channel IS the backpressure: a slow/bursty writer
  stalls producers at send().await instead of growing an unbounded buffer
  (matches the RunHandle mpsc(64) contract the adapters already use).
  - spawn_blocking (not a plain async task): keeps the synchronous rusqlite calls
    off the async runtime's worker threads; the writer returns
    rusqlite::Result<usize> (events written) so a write error stops the drain and
    surfaces on shutdown().await.
  - sender() -> clonable Sender (re-exported as EventSender): one clone per
    producer — a per-run adapter-forwarding task (M3) plus the app's worktree
    watcher (M2/M3). Writer stops when the last sender drops. shutdown() drops
    the log's own sender, awaits the writer, returns the count.
  - KEY BOUNDARY: the store stays AGNOSTIC to the event type. LogEntry carries
    { owner_id, event_json: String } — the already-encoded {"kind":…} JSON that

- crates/core/src/supervisor.rs — RunState: the lifecycle state machine

  runs.status text tokens the store persists; transition() validates edges so an

- crates/core/src/supervisor.rs — RunProcess: per-run process-group spawning +
  group-wide SIGTERM. spawn() sets process_group(0) so the child leads a fresh
  group (pgid == pid); terminate() killpg(pgid, SIGTERM) hits the WHOLE group,
  so a child the agent forked (a shell, a language server) dies too — not just
  the direct child. wait() reaps. This is the PRD's "SIGTERM that group at the
  next iteration boundary" mechanism. Unix-only (cfg via the macOS-only v1
  target; the process/sandbox layers are the platform seam). Uses tokio's
  inherent Command::process_group + libc::killpg (std can't signal a group).
  core/Cargo.toml gains tokio (process) + libc; dev-deps add tokio time/io-util/
  rt-multi-thread.

  wait'`, reads the backgrounded sleep's pid, terminates, then asserts the
  grandchild is gone via kill(pid,0)==ESRCH — proving the signal reaches forked
  descendants, the reason the group exists. + 3 state-machine unit tests.
