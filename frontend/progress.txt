
2026-07-12 — Workbench task 5: per-tab command bar (CommandBar)
- frontend/src/components/CommandBar.tsx: new presentational per-tab command bar
  (the DB client's per-tab command row). Renders, left→right: an object-name
  pill (the task), a `WHERE …`-style client-side event filter, then a right
  cluster with a live "Xs ago" freshness stamp, an agent "Connected"/"missing"
  status pill (from agent_status.installed), and a Run/Re-run action slot
  (children). Every slot is optional so the same component serves a task tab (no
  event stream → no filter/freshness) and a run tab (has both). A small
  Freshness subcomponent ticks once a second against a `since` epoch-ms stamp;
  ago() renders "just now"/"Ns ago"/"Nm ago". No new command surface — consumes
  the existing agent_status only.
- frontend/src/commandbar.css: new stylesheet (imported in main.tsx after
  tabs.css, matching the tab-strip task's pattern). Structural only; all values
  from tokens.css. Task pill (surface-3 chip, ellipsised), monospace filter
  input with a WHERE-style placeholder, right cluster, and on/off agent pills
  (ok-green / danger-red) with a dot.
- frontend/src/components/TaskTab.tsx: relocated the launch control into a
  CommandBar at the top of the tab (task pill + agent status pill for the launch
  default + the existing LaunchControl as the action). The status badge, plan
  path, review banner, and task text now sit below the bar; the run-count
  "compare" button stays in task-tab__actions (shown only when run_count > 0).
  The agent pill names the resolved launch default (settings.default_agent, else
  first installed, else the raw default so an all-missing setup still names one)
  and is Connected iff that agent is installed.
- frontend/src/components/LiveRunView.tsx: added the CommandBar between the run
  header and the body. New client-side `WHERE …` event filter (state `filter`)
  narrows the streamed event list via an eventText() matcher (event kind + its
  payload: text/name/excerpt/cmd/reason/path); an empty-after-filter state shows
  "No events match …". A `lastAt` stamp (reset on run change, bumped on every
  incoming event) feeds the command bar's "Xs ago" freshness while the run is
  active. Fetches agent_status once for the Connected/missing pill on run.agent
  (gated on load so a working agent never flashes "missing"). Moved the task
  text out of the header ident (now in the command bar's pill) to avoid
  duplication; run-view__task CSS stays (RunTimeline still uses it).

  and left PlanView's inline per-task LaunchControl untouched — ripping a working
