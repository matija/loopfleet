# Product

## Register

product

## Users

Developers supervising looping coding agents across local software projects. They need to understand tasks, launch one or more runs, follow agent progress, and compare outcomes without losing context.

## Product Purpose

Loopfleet provides a focused desktop control surface for running and supervising coding agents in sandboxed worktrees. Success means users can scan active work, understand run state, and act quickly without visual noise obscuring the workflow.

## Brand Personality

Quiet, precise, utilitarian. The interface should feel like a mature database client: restrained, dense where useful, and visually consistent.

## Anti-references

Avoid cluttered dashboards, decorative SaaS styling, terminal cosplay, gratuitous cards, saturated colors without semantic meaning, hover effects that move surrounding content, awkward custom form controls, and raw markup leaking into user-facing text.

## Design Principles

1. Keep the current task and its next action obvious.
2. Prefer compact, familiar controls over decorative containers.
3. Use color only for selection, action, and meaningful status.
4. Preserve spatial stability during every interaction.
5. Render user-facing content as readable text, not implementation syntax.

## Accessibility & Inclusion

Target WCAG 2.2 AA. Maintain keyboard-visible focus, sufficient text and control contrast, reduced-motion support, and interactions that never depend on color or hover alone.
